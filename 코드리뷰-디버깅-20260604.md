## 0. 한 장 요약

자동화 테스트는 전부 통과하고 큐·캐시·토큰·암호화 등 인프라 설계 감각은 좋습니다. 하지만 **테스트가 검증하는 영역 바깥에 실거래 사고로 직결되는 결함이 다수** 남아 있습니다. 가장 시급한 5가지는 다음과 같습니다.

1. **로그인 우회가 운영 코드에 살아 있음** (`app.html` L3790, 인증 체크 주석 처리) — 무인증 주문 가능
2. **`_currentUserId` 전역 경합** (`proxy-server.js` L20) — 멀티유저 시 **타인 계좌로 주문/설정 저장** 가능
3. **WebSocket 무수신 watchdog 부재** (`kis-realtime.js`) — 얼어붙은 가격으로 **손절이 영원히 안 걸리는** 시나리오
4. **XSS 무방비** (`app.html`, 이스케이프 0건) — 종목명·뉴스·에러 메시지 경유 계좌 탈취 동급
5. **매수/매도 단건 주문에 확인창 없음 + 이중제출 가능 + 수량 검증 부실** (`app.html`)

자동화 테스트가 손대지 않은 **미체결 추적·날짜 경계·멀티유저·프론트 안전**이 공통 사각지대입니다.

## 1. CRITICAL — 가동 전 반드시 수정

### C-1. 로그인 우회가 운영 코드에 남음 — `app.html` L3790~3800
인증 체크가 주석 처리되어 누구나 앱 진입 → 주문/취소/자동매매 시작 가능.
```js
const me = await checkAuth();
// if (!me) { $('authScreen')...; return; }   ← 주석 처리됨
$('app').style.display = 'grid';              ← 무조건 진입
```
**수정**: 주석 4줄 원복. 배포 체크리스트에 "인증 가드 동작 확인" 추가.

### C-2. `_currentUserId` 전역 경합 → 타 유저 계좌 혼선 — `proxy-server.js` L20~21, L899
요청 핸들러가 `async`이고 곳곳에 `await`가 있는데, 유저 식별을 전역 변수 한 개로 관리. A의 요청이 `await` 중 B의 요청이 들어오면 `_currentUserId`가 B로 바뀌어, A 재개 시 `loadConfig()/saveConfig()`가 **B의 앱키·계좌를 읽고 씀**. 토큰도 엉뚱한 유저 파일에 기록.
**수정**: 전역 변수 제거. 핸들러 진입 시 `const cfg = loadConfigFor(userId)`로 요청 스코프 고정, `saveConfigFor(userId, cfg)` 명시 전달(또는 `AsyncLocalStorage`).

### C-3. WebSocket 무수신 watchdog 부재 → 얼어붙은 가격 — `kis-realtime.js`
TCP half-open(NAT 타임아웃·절전·단선) 시 `close` 이벤트가 안 와서 `connected=true`인 채 시세가 멈추고, 재연결은 close에만 의존하므로 발동 안 함. 엔진이 이 가격 캐시를 쓰면 **손절 판정이 영원히 멈춤**.
**수정**: 마지막 수신 시각 기록 + 60~120초 watchdog로 무수신 시 `sock.destroy()` 강제 재연결.

### C-4. XSS — 외부 데이터 무이스케이프 innerHTML (15곳+) — `app.html`
이스케이프 함수가 **파일 전체에 0개**. 종목명(`prdt_name`/`hts_kor_isnm`), 뉴스 제목·`href`(L2514, 2944), 자동매매 로그(L3042, 3486), 검색결과(L1394), 서버 에러 토스트(L1308)가 그대로 innerHTML로 삽입. API 키·주문 권한을 가진 세션이라 XSS = 계좌 탈취.
**수정**:
```js
const esc = s => String(s??'').replace(/[&<>"']/g,c=>({'&':'&amp;','<':'&lt;','>':'&gt;','"':'&quot;',"'":'&#39;'}[c]));
```
모든 보간 지점 적용. 뉴스 링크는 `^https?://` 검증 후에만 `<a>` 생성.

### C-5. 매수/매도 주문 안전장치 부재 — `app.html` `submitOrder`(L2211)·`dashOrder`(L1906)·`submitSideOrder`(L3065)
취소·분할주문·자동매매에는 `confirm`이 있는데 **단건 매수/매도는 클릭 즉시 발사**. 게다가 inflight 동안 버튼 비활성화/플래그가 없어 **연타 시 동일 주문 N건 접수**. 수량은 `"0"`·`"-3"`·`"1.5"`가 통과하고, 가격 `0` 입력 시 **지정가→시장가 암묵 전환**(`orderType: price>0?'00':'01'`).
**수정**: ① 매수/매도 confirm 추가(실전모드 시 ⚠️ 강조), ② `_orderInflight` 가드 + 버튼 disable, ③ `const q=parseInt(qty,10); if(!Number.isInteger(q)||q<1) return;` + 호가단위 보정.

### C-6. `unit` 미검증 → 무한루프로 서버 정지 — `proxy-server.js` L1282 (분봉)
`const unit = parseInt(query.unit||'5')`만 하고 범위 검증 없음. `unit=0` 전달 시 `barsPerDay = Infinity`, 캔들 보간 루프 `i += 0`이 **무한루프 → 이벤트 루프 정지 → 서버 전체 행**. 요청 한 번으로 다운 가능.
**수정**: `const unit = Math.max(1, Math.min(60, parseInt(query.unit)||5));`

## 3. 알려진 과제 4건 — 검증 결과

**① 주문 지연(22.5초)**: 큐 우회 + 고정 600ms로 "재시도×큐 대기 중첩" 구조는 제거됨. 단 ⓐ `holdUntil`이 inflight를 못 막아 동시 주문 충돌 재발 가능(H-8), ⓑ 토큰 만료 겹치면 65초 쿨다운에 걸려 **주문 1분간 불가** 경로 잔존. 권장: 600ms 고정 대기를 "마지막 발사 후 경과" 가변 대기로, 타임아웃 시 1회 한정 재시도(미접수 확인 후).

**② 체결 확인(접수≠체결)**: 구현은 됨(잔고 reconcile + WS 체결통보 + 10분 자동취소). 결함 2건 — ⓐ `reconcile(null,…)` 크로스유저 오염(H-9), ⓑ **부분체결 미처리**: reconcile은 전량 기준이라 부분체결은 영원히 '접수'로 남고, markCancel이 status를 '취소'로 덮어 **체결 이력 소실**. 동일 종목 pending 2건이면 잔고 1회 증가로 둘 다 체결 오판.

**③ 시장현황 종목 코드 표시**: 프론트 폴백 체인(`hts_kor_isnm || nameOf(code)`, L3130)은 정상. 근본 원인 두 겹 — ⓐ `nameOf`(L1339)가 미등록 코드를 **코드 그대로 반환**, STOCK_DB가 ~160종목뿐, ⓑ **서버 `/api/volume100` 폴백 경로에서 `acml_vol`을 캐시 안 함**(`refreshPrice`가 price/chgPct/sign/prev만 저장, L690) → 거래량 상위가 **거래량 0으로 정렬**되고 이름도 누락. **수정**: `refreshPrice`에서 `data.accVol = parseInt(o.acml_vol||0)` 저장 + 서버가 코드→이름 사전 제공.

**④ market-overview 종목명**: ③과 동일 원인.

## 5. 테스트 커버리지 갭 (프로덕션에서 터지는데 테스트 0건)

엔진: **미체결 후 잔고 재등장→중복 매도**(가장 위험), placeOrder 예외 시 쿨다운, 주문 거부(rt_cd≠0) 경로, **dailyLossLimit 도달→자정 KST 리셋**, **MA 교차 자체**(RSI만 테스트), 시간 경계(15:20 컷오프), 공휴일.
실시간: **TCP 분할 프레임 파싱**, **무수신 데드 커넥션**, PINGPONG 미응답, LRU가 활성 구독 죽이는지, 재연결 시 체결통보 재구독, 멀티 SSE 콜백 덮어쓰기.
인증: **테스트 파일 자체 없음** — register/login/만료/동시쓰기/decrypt 실패 전부 0건.
저널: 부분체결, 동일종목 다중 pending 오판, userId 누락 매칭, markFilled.
