6개 영역 심층 병렬 리뷰(반응속도 / SQLite 계층 / 자동매매 엔진 / 프론트 / 서버코어 / 실시간·운영)
→ 회귀·병목 수정 → 적대적 재검증 → 회귀 테스트 추가. **테스트 120 → 131개 전부 통과.**

## 🔴 데이터/서버 즉사급 (수정 완료)
- **저널 markFilled/markCancel가 userId를 LIMIT 뒤에 걸러** 동일 odno(유저 간 충돌 가능) 다건에서
  체결 누락·유령 미체결 → userId를 WHERE로 이동. (SQLite 전환 회귀)
- **unhandledRejection 핸들러 부재** → 가드 밖 reject 한 건이 멀티유저 서버 즉사 → 핸들러 추가.
- DB busy_timeout=5000 (서버↔market-check 동시 접근 시 로그인 500 방지).

## 🟠 자동매매 위험관리 (수정 완료)
- **부분체결 매수 후 잔여취소분을 '매도'로 오계상** → 유령 손익이 일일손실·연속패 서킷 오염.
  실현손익을 "실제 낸 매도분(_sellPending)"에만 계상하도록 변경.
- _sellPending 누수(지정가 미체결 취소) → 6분 경과 시 정리(수동매도 오귀속 방지).
- 시장가 손절 타임아웃-접수 시 손실 누락 → 시장가는 예외에도 기록해 잔고대조에서 계상.
- rt_cd=0 빈 output1(일시 오류)이 봇 지분 통째 삭제 → 1틱 유예 후 재확인. 시간설정 NaN 검증.

## ⚡ 반응속도 (첫 클릭 ~2초 콜드 → 0 목표)

- 선페인트(시세표 가격을 클릭 즉시 헤드라인 반영), refreshBuyable 병렬화+seq가드.

- _minDayCache·SWR 캐시 KST 자정 prune(메모리 누수 차단).

## 🔒 서버 보안 / 프론트 안전 (수정 완료)
