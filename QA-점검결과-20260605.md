# 전수 QA/QC 점검 결과 (2026-06-05)

원격 공개 배포를 위한 전수 점검·수정·검증 기록.

## 🐛 발견·수정한 버그 (실제 원인)

| # | 문제 | 영향 | 수정 |
|---|------|------|------|
| 1 | **`PROXY='http://localhost:3000'` 하드코딩** (app.html) | 원격/터널 접속 시 모든 API가 *사용자 본인 PC*로 감 + HTTPS 혼합콘텐츠 차단 → **회원가입·로그인 등 전부 실패** (페이지만 뜸) | `PROXY=''` (same-origin 상대경로) |
| 2 | **로그아웃 500** (`cookies` 변수 스코프 밖) | 로그아웃 불가 | 핸들러에서 직접 파싱 |
| 3 | **가입 IP제한이 터널 뒤에서 전체 합산** | 터널 사용자 전원이 `127.0.0.1`로 보여 3회 후 모두 차단 | 실제 IP(`CF-Connecting-IP`) 인식 |
| 4 | 가입 한도 3회 → 과도 | NAT 공유 사용자 차단 | 10회로 완화 |
| 5 | KIS 미설정 시 **503 에러** | 키 입력 전 대시보드 콘솔이 에러로 도배 | 200 + `needConfig` 플래그 |
| 6 | 본문 크기 무제한 | 메모리 DoS | 256KB 상한 |
| 7 | 멀티유저 체결 교차오염 (`reconcile(null)`) | A 잔고로 B 주문 오판 | `reconcile(userKey)` |
| 8 | 에러 메시지 원문 노출 / export `__userId` 노출 | 정보 누출 | 일반화 / 필드 제거 |
| 9 | 쿠키 `Secure` 없음 | HTTPS에서 탈취 위험 | HTTPS 뒤 자동 부여 |

## ✅ 검증 (실제 터널 통해)

- **인증 전 과정**: 가입→admin부여→세션유지→로그아웃→재로그인→오답거부 **모두 정상**
- **전체 API 16종**: KIS 키 없이도 5xx 0건, 모두 graceful(`needConfig`)
- **자동매매**: 상태/로그/설정/시작/정지 동작 (키 없을 때 start는 안내 후 거부)
- **보안**: 소스파일(.js)·민감파일(users.json/.enckey)·경로 traversal **전부 차단**(파일 내용 노출 0)
- **단위 테스트 71개 전부 통과** (모든 수정 후에도)
- **자가복구**: 터널 강제 종료 시 감시 스크립트가 새 터널 1개로 자동 복구 + URL 자동 갱신 확인

## 🚀 운영 구조

- `serve.sh` : 감시(supervisor). 서버+터널을 단일로 유지, 죽으면 재기동. 락파일로 중복 실행 차단.
- `run-tunnel.sh` : Cloudflare 터널 1개 실행 + 공개 URL을 `CURRENT-URL.txt`에 기록.
- `show-url.sh` : 현재 공개 주소 출력. **URL이 바뀌어도 이걸로 항상 확인.**
- 서버는 `127.0.0.1`만 바인딩(직접 노출 차단), 외부는 HTTPS 터널로만.

### 수동 제어
```bash
./show-url.sh                       # 현재 주소 확인
nohup setsid bash ./serve.sh &      # (꺼졌을 때) 다시 시작
pkill -f serve.sh; pkill -f run-tunnel.sh; pkill -f 'cloudflared tunnel'; pkill -f proxy-server.js  # 전체 중지
```

## ⚠️ 남은 한계 (출시 전 참고)
- quick tunnel은 **재시작 시 주소가 바뀜** (고정 주소는 무료 Cloudflare 계정+도메인 필요 → `DEPLOY.md` 참고). 그동안은 `show-url.sh`로 현재 주소 확인.
- 잘못된 KIS 키 입력 시 가격조회가 generic 500 (설정 화면에서 토큰 발급 실패로 별도 안내됨).
- auth.js 단위 테스트 없음(수동 E2E로 검증함).
