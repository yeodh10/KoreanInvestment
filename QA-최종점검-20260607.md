## 🔴 CRITICAL (출시 차단 → 해결)

1. **매도 접수=체결 오인 + 자동취소 시 포지션 고아화** (`auto-trader.js`)
   - 매도를 '접수' 즉시 봇 지분 차감·실현손익 확정 → 묵은 지정가가 미체결→10분 후 자동취소되면
     주식은 계좌에 남는데 봇 장부에서 사라져 **손절 관리가 영구 중단**.
   - 수정: 봇 지분 축소·실현손익을 **잔고 대조(실제 체결) 시점으로 일원화**. 손절/트레일은 **시장가**로 전환.
     → 고아화·부분체결 오삭감·헤드리스 미확정을 한 번에 해결.

2. **손절/트레일이 최대 5분 묵은 가격으로 판정** (`auto-trader.js`)
   - 보유 관리 루프가 잔고의 `prpr`(5분 주기)만 사용 → 급락장 손절 지연, `prpr` 누락 시 cur=0 허위 손절.
   - 수정: 보유 종목은 **실시간 현재가 재조회** 후 판정, `cur<=0`이면 skip.

3. **실시간 WS가 "열린 적 없는 연결" 실패 시 영구 정지** (`kis-realtime.js`)
   - 핸드셰이크 전 실패(ECONNREFUSED·거부·타임아웃)에서 `onclose` 미발화 → `connecting` 고착 →
     이후 모든 재연결 무시 → 장중 실시간 전체 사망.
   - 수정: MiniWS가 open 여부와 무관하게 **종료 콜백 1회 보장** + 연결/approval **타임아웃** 추가.

4. **실전 계좌인데 주문 확인창이 "[모의투자]" 표시** (`app.html`)
   - `_txModeCached`가 거래 페이지 진입 시에만 채워짐 → 대시보드에서 바로 주문하면 실전이 모의로 표시.
   - 수정: 부팅 `checkProxy`에서 모드 캐시. 미확인 시 "⚠️ 모드 미확인(실전일 수 있음)" 경고.

## 🟠 HIGH

- **잔고 `rt_cd` 미검사** → 오류 응답 1회로 전 포지션 무관리화. → `rt_cd==='0' && Array.isArray(output1)` 가드.
- **설정 NaN fail-open** → `"NaN"`/Infinity 주문. → `_clampSafety` 범위 강제 + 수량 `Number.isFinite` 가드.
- **헤드리스(브라우저 미접속) 체결 미확정** → 엔진이 잔고 대조 시 `orderJournal.reconcile` 직접 호출.
- **부분체결 중 봇 지분 오삭감** → 미체결 매수 잔량(`buyQty`)을 더한 "체결 시 도달 수량"으로 대조.
- **주문 12초 타임아웃 → "실패" 오표시 → 재클릭 중복주문** → abort를 `_timeout`으로 구분, "응답 지연·확인" + 재조회.
- **`htsId` 미저장** → 멀티유저 체결통보 기능 침묵 사망. → `auth.saveUserConfig/loadUserConfig`에 추가.
- **멀티탭 체결통보 N배 중복 집계** → 피드당 단일 `_journal` 훅(탭 0개여도 유지 → 유실도 방지).
- **세션 해시 폴백 무력화** → `sessions[token]` 원본키 폴백 제거(유출 시 무력화 방지).
- **`submitSplitOrder` 검증·모드·중복가드 우회** → `validateOrder`+`confirmOrder`+`_orderInflight` 적용.
- **`loadStock`/탭 로더 레이스** → 시퀀스 가드(늦은 옛 응답이 새 종목 화면 덮기 방지).
- **인라인 onclick JS-문자열 XSS** → `jsArg()` 도입(`'`·`\`·`<`를 \u 이스케이프), cancel/sellFromPf/watchChip 적용.
- **UX**: `.chip` CSS 깨짐, 알림폼/백테스트통계 `display:none` 무시(중복 style), 매도 "전량"이 예수금 기준,
  모바일 미대응(@media 0건), 라벨 대비 미달, 실전 모드 파란 배지.
